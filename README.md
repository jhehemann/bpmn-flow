# bpmn-flow

Toolkit for iteratively authoring BPMN 2.0 diagrams with an AI coding assistant (Claude Code, Codex CLI, …). You issue voice or text instructions, the assistant edits the semantics in `flows/*.bpmn`, a layout script regenerates the diagram interchange, and a lightweight browser viewer renders the result live (polling every ~1.5 s).

## Quick Start

```bash
git clone https://github.com/jhehemann/bpmn-flow.git
cd bpmn-flow
npm install
python3 -m http.server 8000   # in a second terminal
open http://localhost:8000
```

The viewer shows `flows/example.bpmn`. Edit the file (or let the AI assistant edit it), run `npm run layout && npm run validate` — the browser auto-reloads.

> **Building your own?** Click **"Use this template"** on GitHub to create a fresh repo under your account, then clone *that* URL instead.

## Concept

- **Semantics decoupled from layout**: The assistant only edits the semantic parts of the BPMN file. The layout (coordinates, waypoints, bounds) is regenerated by `npm run layout`. The assistant never has to compute positions, and PR diffs surface semantic changes instead of layout noise.
- **Browser viewer as live preview**: `index.html` renders the selected diagram and polls every ~1.5 s. Editor-agnostic and reloads reliably.
- **VS Code extension as optional add-on** for visual modeling via "Open With" (see below).
- **Validation**: `bpmnlint` checks conventions (default flow, IDs, connectivity) before commit or in CI.

## Requirements

- [Node.js](https://nodejs.org/en/download) ≥ 18 (or via [nvm](https://github.com/nvm-sh/nvm#installing-and-updating))
- Any static web server for the viewer — `python3 -m http.server` works; alternatives: `npx serve`, `caddy file-server`, etc.
- A modern browser
- An AI coding assistant — developed against [Claude Code](https://docs.claude.com/en/docs/claude-code); other harnesses like [Codex CLI](https://github.com/openai/codex) should work analogously.
- **Optional**: VS Code/Cursor with the [`bpmn-io.vs-code-bpmn-io`](https://marketplace.visualstudio.com/items?itemName=bpmn-io.vs-code-bpmn-io) extension for visual modeling. VS Code suggests the install automatically when you open the repo.

## Iterative workflow

```
instruct → edit semantics → npm run layout → npm run validate → commit
```

You issue an instruction, the assistant edits `flows/<file>.bpmn`, then runs layout and validate itself (see `CLAUDE.md`). The browser viewer reloads within ~1.5 s; you inspect, refine, and commit when an intermediate state is solid.

## NPM scripts

| Script              | Purpose                                                              |
|---------------------|----------------------------------------------------------------------|
| `npm run layout`    | Auto-layout for all `flows/*.bpmn`. Optionally targeted: `npm run layout -- flows/foo.bpmn`. |
| `npm run validate`  | `bpmnlint` over all `flows/*.bpmn`. Configuration in `.bpmnlintrc`.  |

## Repository layout

| Path                       | Purpose                                                |
|----------------------------|--------------------------------------------------------|
| `flows/*.bpmn`             | BPMN 2.0 diagrams — one per workflow                   |
| `index.html`               | Browser viewer with multi-file picker and auto-reload  |
| `scripts/layout.mjs`       | Auto-layout wrapper (`bpmn-auto-layout`)               |
| `.bpmnlintrc`              | Lint rules                                             |
| `.vscode/extensions.json`  | Recommendation for the bpmn-io modeler extension       |
| `.vscode/settings.json`    | Workspace settings (open `.bpmn` as text by default)   |
| `CLAUDE.md`                | Edit rules and conventions for the AI assistant        |
| `package.json`             | Tooling dependencies (not runtime-relevant)            |

## Other assistants than Claude Code

`CLAUDE.md` holds the rules the assistant must follow when editing. The rules are assistant-agnostic — only the file name differs:

- **Codex CLI** → `AGENTS.md`
- **Cursor** → `.cursor/rules/bpmn.mdc`
- **Aider** → `CONVENTIONS.md` (load with `aider --read CONVENTIONS.md`)

Recommendation: symlink instead of copying (`ln -s CLAUDE.md AGENTS.md`).

## Team best practices

- **Small commits.** Each clearly delimited intermediate state gets its own commit. Easier to revert and review.
- **Maintain `CLAUDE.md` actively.** If the assistant repeatedly violates a convention, codify the rule there instead of repeating it in prompts.

## Visual editing (optional)

`.bpmn` files open as text in VS Code by default. For drag-and-drop editing, **right-click → "Open With…" → "BPMN Editor"** (extension `bpmn-io.vs-code-bpmn-io`).

### Important: don't work in parallel with the assistant

The visual editor's WebView caches the file state at open time and overwrites external changes on the next save (manual `Cmd+S` or active `files.autoSave`). Quick manual tweaks are fine, but follow the rules:

1. **Pause the assistant before opening the editor**, then use "Open With".
2. **Close the editor tab when done** — an open tab silently overwrites on the next focus change.
3. While the assistant is editing, use the browser viewer at `http://localhost:8000` — it reloads race-free via polling.

## Known limitations

**Auto-layout is destructive.** `npm run layout` regenerates the entire `<bpmndi:...>` block from the semantics. Position tweaks from the visual editor are lost on the next assistant edit — deterministic diffs trade fine-grained layout control for clean PRs.

## Troubleshooting

| Symptom                                  | Cause and fix                                                                                |
|------------------------------------------|----------------------------------------------------------------------------------------------|
| Viewer shows `Error: …`                  | Syntax or structural error in `flows/*.bpmn`. Open the browser console for details.          |
| Diagram doesn't refresh in the browser   | Is the auto-reload checkbox on? Is the server running? Try a hard reload (`Cmd+Shift+R`).    |
| `npm run validate` fails                 | A lint rule was violated (e.g. unconnected element). The error message names element + rule. |
| Edge labels overlap with lines           | Known weakness of `bpmn-auto-layout`. Workaround: shorter labels or accept it.               |
| BPMN editor shows a stale state          | Close the tab and reopen via "Open With", or Cmd+Shift+P → "Developer: Reload Window".       |
